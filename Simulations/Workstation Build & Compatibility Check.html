Mentor: ON

How to Use This Simulation

Objective

You've got two work orders today, each for a different end user with different needs. Read the requirements for each build carefully — nothing here is interchangeable between tickets.

How to Use This Simulator

  1. Work Order 1 is active first. Choose every component (CPU, RAM, GPU, Storage, Network, Peripherals) from the dropdowns.
  2. Watch the meter update live — it reflects your choices before you submit anything.
  3. Click Submit Build for Debrief for a pass/fail verdict with reasoning.
  4. Work Order 2 stays locked until Work Order 1 passes.
  5. Use Mentor: ON/OFF to show or hide live hints while you shop for parts. Turn it off to test yourself cold.
  6. New Ticket Pair draws two different work orders at random from a larger pool — run it again and you likely won't see the same pairing twice.

Why the requirements matter

Every "wrong" choice below maps to a real compatibility or right-sizing concept: CPU architecture and OS support, integrated vs. dedicated graphics, SSD vs. HDD, matching RAM to workload, and wired vs. wireless reliability. Focus on why a choice fails, not just which one does.
