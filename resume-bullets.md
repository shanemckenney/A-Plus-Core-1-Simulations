# Resume Bullet Examples — CompTIA A+ Simulation Lab

Each bullet below is built on Google's **XYZ format** — accomplishment, result, method — but written as a normal sentence instead of a fill-in-the-blank template:

> [Action verb + what you did] + [result/metric] + [how you did it]

Pick the bullet that best matches the role you're applying for (IT Support/Help Desk vs. technical/dev-adjacent roles), then swap in a real number from your own experience — a score, a time saved, a percentage improved — before putting it on an actual resume.

---

## Tier 1: Must-Know

### Workstation Hardware Troubleshooting (Domain 5.1)
- Diagnosed and resolved simulated motherboard, RAM, CPU, and power supply failures with a 90%+ first-pass accuracy rate, applying a structured troubleshooting methodology (theorize, test, verify, document) to each case.
- Cut average diagnosis time on hardware failure scenarios, including POST beep codes and thermal issues, by systematically isolating power, connection, and component variables before recommending replacement parts.

### SOHO Wireless Router GUI Setup (Domain 2.6)
- Configured DHCP scopes, static IP addressing, and WPA2/WPA3 security through router web GUIs, achieving full connectivity and secured Wi-Fi access across every test scenario.
- Hardened simulated SOHO networks by eliminating default credentials, open SSIDs, and outdated firmware settings, closing every common configuration vulnerability identified.

### Workstation Build & Compatibility Check (Domain 3.5)
- Verified CPU/motherboard socket and chipset compatibility across custom builds with a zero-error rate, cross-referencing socket types, RAM specifications, and BIOS/UEFI requirements before assembly.
- Configured BIOS/UEFI settings — boot order, TPM, Secure Boot — on every test build to meet security and functionality requirements.

### Mobile Device Troubleshooting (Domain 5.4)
- Triaged mobile device hardware issues — batteries, screens, charging ports, connectivity — correctly calling repair-vs-replace decisions on every practice case by weighing symptoms against manufacturer failure patterns and safety guidelines.
- Resolved liquid damage and overheating cases using a structured risk-assessment process that prioritized user data and device safety at every step.

### MFD Printer Deployment & Troubleshooting (Domain 3.7)
- Deployed multifunction printers across simulated network environments with successful driver installation and connectivity on every unit, selecting the correct driver type (PCL vs. PostScript) and configuring network, USB, and wireless connections.
- Enabled user authentication and access restrictions on every simulated device during deployment, aligning printer security configurations with organizational policy.

## Tier 2: Deeper Learning

### Ports & Protocols Matching (Domain 2.1)
- Built working fluency with 20+ common TCP/UDP ports and protocols (FTP, SSH, SMTP, DNS, HTTP/HTTPS, RDP, and more), correctly matching every port to its service in timed practice rounds.
- Cut identification time on protocol-matching tasks by memorizing port number ranges and their associated service categories — email, remote access, web, file transfer.

### Power Protection Sorter (Domain 3.6)
- Matched power supply specifications to real-world use cases, correctly pairing voltage, wattage, and form factor requirements across every sorting exercise by comparing input voltage ranges, modular design, and efficiency ratings.
- Selected appropriate UPS/surge protection tiers for varied deployment scenarios by weighing runtime, load capacity, and cost trade-offs.

### Laser Printer Troubleshooting (Domain 3.8)
- Performed toner replacement and maintenance kit installations across every laser printer practice module, following manufacturer-specified service intervals and cleaning procedures for optical components.
- Diagnosed print quality issues — streaking, ghosting, color calibration — by isolating the failing stage of the imaging process (charging, exposing, developing, transferring, fusing).

### Audio Pathway Troubleshooting (Domain 1.1)
- Diagnosed mobile device audio hardware failures, correctly distinguishing software from hardware causes by isolating the responsible pathway — microphone, speaker, Bluetooth stack, or physical connector.
- Resolved Bluetooth connectivity and audio jack issues on smartphones and tablets with a 100% correct-resolution rate, using a layered troubleshooting approach (pairing, OS settings, hardware).

### Diagnostic Bench: Multi-Case Simulator (Multi-Domain)
- Diagnosed and resolved four independent hardware/network trouble tickets — power budgeting, thermal failure, CPU overclock instability, and cable-level network faults — achieving correct root-cause and remediation on every case by calculating system power draw against PSU capacity and cross-referencing BIOS/UEFI telemetry.
- Ran a live command-line diagnostic workflow (ping, tracert, netstat -e) to isolate a VoIP packet-loss issue to a damaged physical patch cable, ruling out routing and software causes before recommending remediation.

## Tier 3: Specialized

### RAID Storage Configuration (Domain 3.4)
- Selected optimal RAID configurations (0, 1, 5, 6, 10) for varied storage requirements, calculating usable capacity and weighing fault-tolerance trade-offs for each option.
- Modeled storage failure recovery scenarios across RAID arrays, accurately predicting data-loss risk under single- and multi-drive failures by applying RAID parity and mirroring principles.

## Bonus: Acronym Flashcards Study Tool

If you're using this for a software/web development resume rather than (or in addition to) an IT support one, the flashcard app itself is a solid project bullet:

- Built a spaced-repetition study app covering 150+ certification acronyms across three progressively harder phases (recall, application, scenario-based quizzing), implementing a phase-promotion algorithm that advances items based on performance and persists progress via browser localStorage.
- Designed an adaptive quiz engine that filters a 150+ item dataset in real time based on user progress, using JavaScript to render, score, and re-queue flashcards without a backend or database.
