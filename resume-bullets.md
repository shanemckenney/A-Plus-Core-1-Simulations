# Resume Bullet Examples — CompTIA A+ Simulation Lab

Each simulation below has ready-to-use resume bullets written in Google's **XYZ format**:

> "Accomplished **[X]**, as measured by **[Y]**, by doing **[Z]**."

Pick the bullet that best matches the role you're applying for (IT Support/Help Desk vs. technical/dev-adjacent roles), then tweak the numbers and phrasing to reflect your actual experience. Treat the "measured by" clause as a placeholder — swap in a real metric once you have one (e.g., an actual quiz score, time-to-completion, or a percentage of scenarios resolved correctly).

---

## Tier 1: Must-Know

### Workstation Hardware Troubleshooting (Domain 5.1)
- Diagnosed simulated motherboard, RAM, CPU, and power supply failures, as measured by correctly identifying the root cause in 90%+ of practice scenarios, by applying the CompTIA troubleshooting methodology (identify, theorize, test, establish a plan, verify, document).
- Resolved hardware failure scenarios involving POST beep codes and thermal issues, as measured by reducing average diagnosis time per scenario, by systematically isolating variables (power, connections, components) before escalating to replacement parts.

### SOHO Wireless Router GUI Setup (Domain 2.6)
- Configured small office/home office network infrastructure, as measured by achieving full connectivity and secured Wi-Fi access across all test scenarios, by setting up DHCP scopes, static IP addressing, and WPA2/WPA3 security through the router's web GUI.
- Hardened a simulated home network against common vulnerabilities, as measured by closing 100% of default-configuration security gaps (default credentials, open SSID, outdated firmware settings), by applying SOHO security best practices.

### Workstation Build & Compatibility Check (Domain 3.5)
- Validated component compatibility for custom PC builds, as measured by a zero-error rate across CPU/motherboard socket and chipset pairings, by cross-referencing socket types, RAM specifications, and BIOS/UEFI requirements before assembly.
- Configured BIOS/UEFI settings for secure, functional workstation builds, as measured by successfully enabling boot order, TPM, and Secure Boot across all test builds, by methodically working through firmware settings relevant to each hardware configuration.

### Mobile Device Troubleshooting (Domain 5.4)
- Triaged mobile device hardware issues (batteries, screens, charging ports, connectivity), as measured by correctly categorizing repair-vs-replace decisions in all practice cases, by evaluating symptoms against manufacturer failure patterns and safety guidelines (e.g., swollen battery handling).
- Diagnosed liquid damage and overheating scenarios on mobile devices, as measured by identifying the correct next troubleshooting step in each case, by applying a structured risk-assessment process prioritizing user data and device safety.

### MFD Printer Deployment & Troubleshooting (Domain 3.7)
- Deployed multifunction printers across simulated network environments, as measured by achieving successful driver installation and connectivity in 100% of scenarios, by selecting the correct driver type (PCL vs. PostScript) and configuring network, USB, and wireless connections.
- Implemented printer security controls during deployment, as measured by enabling user authentication and access restrictions on every simulated device, by configuring duplex, tray, and security settings aligned with organizational policy.

## Tier 2: Deeper Learning

### Ports & Protocols Matching (Domain 2.1)
- Built fluency with 20+ common TCP/UDP ports and protocols, as measured by matching all ports to their correct service in timed practice rounds, by repeatedly drilling port-to-protocol associations (FTP, SSH, SMTP, DNS, HTTP/HTTPS, RDP, and more).
- Strengthened network troubleshooting speed, as measured by reducing protocol-identification time per question, by memorizing port number ranges and their associated service categories (email, remote access, web, file transfer).

### Power Protection Sorter (Domain 3.6)
- Matched power supply specifications to real-world use cases, as measured by correctly pairing voltage, wattage, and form factor requirements in all sorting exercises, by comparing input voltage ranges (110-120 vs. 220-240 VAC), modular vs. non-modular designs, and efficiency ratings.
- Evaluated power protection options for varied deployment scenarios, as measured by selecting appropriate UPS/surge protection tiers for each use case, by weighing runtime, load capacity, and cost trade-offs.

### Laser Printer Troubleshooting (Domain 3.8)
- Performed preventive maintenance procedures on laser printers, as measured by successfully completing toner replacement and maintenance kit installation in all practice modules, by following manufacturer-specified service intervals and cleaning procedures for optical components.
- Diagnosed print quality issues (streaking, ghosting, color calibration), as measured by correctly identifying the failing component in each scenario, by isolating the imaging process step (charging, exposing, developing, transferring, fusing) responsible for the defect.

### Audio Pathway Troubleshooting (Domain 1.1)
- Diagnosed mobile device audio hardware failures, as measured by correctly distinguishing software vs. hardware causes in all test scenarios, by isolating the audio pathway (microphone, speaker, Bluetooth stack, physical connector) responsible for each symptom.
- Resolved Bluetooth connectivity and audio jack issues on smartphones/tablets, as measured by a 100% correct-resolution rate across simulated cases, by applying a layered troubleshooting approach (pairing, drivers/OS settings, hardware).

## Tier 3: Specialized

### RAID Storage Configuration (Domain 3.4)
- Selected optimal RAID configurations (0, 1, 5, 6, 10) for varied storage requirements, as measured by matching the correct RAID level to redundancy/performance needs in all scenarios, by calculating usable capacity and evaluating fault-tolerance trade-offs for each configuration.
- Modeled storage failure recovery scenarios across RAID arrays, as measured by correctly predicting data-loss risk under single- and multi-drive failure conditions, by applying RAID parity and mirroring principles to each configuration.

## Bonus: Acronym Flashcards Study Tool

If you're using this for a software/web development resume rather than (or in addition to) an IT support one, the flashcard app itself is a solid project bullet:

- Built a spaced-repetition study application covering 150+ certification acronyms, as measured by supporting three progressively harder study phases (recall, application, scenario-based quizzing), by implementing a phase-promotion algorithm that advances items based on user performance and persists progress via browser localStorage.
- Designed an adaptive quiz engine with scenario-based multiple choice questions, as measured by dynamically filtering a 150+ item dataset per user progress in real time, by writing JavaScript logic to render, score, and re-queue flashcards without a backend or database.

*Tip: Google's XYZ format works best when the "Y" (measurement) is a real number from your own experience — a score, a time saved, a percentage improved. Use the placeholders above as a starting structure, not literal claims.*
